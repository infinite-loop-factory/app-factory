# 로또 번호 추첨 기능 PRD

## 1. 개요

로또 번호 추첨 기능은 사용자가 버튼을 누르거나 기기를 흔들어 한국 로또 6/45 번호 1게임을 빠르고 재미있게 추천받는 기능이다. v1은 "로또볼 추첨기" 경험에 집중하며, 번호 저장이나 당첨번호 조회처럼 네트워크 및 장기 데이터가 필요한 기능은 포함하지 않는다.

### 목표

- 사용자가 홈에서 로또 기능으로 진입해 즉시 번호 추첨을 시작할 수 있다.
- 1부터 45까지 중복 없는 숫자 6개를 클라이언트에서 생성한다.
- 번호는 로또볼이 섞이고 순서대로 뽑히는 연출을 통해 공개된다.
- 최종 결과는 추첨된 순서를 유지한다.
- 결과가 무작위 추천이며 당첨을 보장하지 않는다는 안내를 명확히 제공한다.

### 비목표

- 생성 기록 저장
- 즐겨찾기 저장
- 최근 또는 회차별 당첨번호 조회
- 생성 번호와 당첨번호 비교
- 외부 API 또는 네트워크 연동
- 회원/계정 기능
- 사운드 효과
- 로또 6/45 외 다른 복권 지원

## 2. 사용자 경험

### 진입

- 홈(`/`) 기능 허브에 "로또 번호 추첨" 카드가 추가된다.
- 사용자가 해당 카드를 선택하면 `/lotto` 화면으로 이동한다.
- `/lotto` 화면은 네이티브 Stack 헤더를 사용하며, 기존 `/food` 화면처럼 홈으로 돌아갈 수 있어야 한다.

### 초기 상태

- 화면은 아직 추첨 결과가 없는 상태로 시작한다.
- 사용자는 추첨 버튼을 눌러 번호 추첨을 시작할 수 있다.
- 네이티브 환경에서는 기기를 흔들어 추첨을 시작할 수 있음을 안내한다.
- 웹 환경에서는 흔들기 안내를 노출하지 않거나, 버튼으로만 추첨할 수 있음을 명확히 한다.

### 추첨 중 상태

- 사용자가 버튼을 누르거나 네이티브 환경에서 기기를 흔들면 추첨이 시작된다.
- 추첨 중에는 중복 실행을 막기 위해 추첨 버튼이 비활성화된다.
- 로또볼 추첨기 콘셉트의 애니메이션을 보여준다.
- 6개 번호는 한 번에 모두 나타나지 않고 순서대로 공개된다.
- 번호가 공개될 때 또는 추첨이 완료될 때 네이티브 환경에서 가벼운 햅틱 피드백을 제공한다.
- 웹 환경에서는 햅틱 없이 시각적 연출만 제공한다.

### 결과 상태

- 추첨이 완료되면 6개 번호를 추첨순으로 표시한다.
- 결과 번호는 1부터 45 사이의 중복 없는 숫자여야 한다.
- 결과 영역에는 다음 취지의 안내 문구를 표시한다: "무작위 추천이며 당첨을 보장하지 않아요."
- 사용자는 다시 추첨 버튼을 눌러 새 번호를 생성할 수 있다.
- 다시 추첨하면 이전 결과는 화면 상태에서 새 결과로 대체된다.

## 3. 기능 요구사항

### 번호 생성

- 클라이언트에서 번호를 생성한다.
- 생성 범위는 1 이상 45 이하의 정수다.
- 생성 개수는 정확히 6개다.
- 한 조합 안에서 숫자는 중복될 수 없다.
- 최종 표시 순서는 오름차순 정렬하지 않고 추첨된 순서를 유지한다.
- 생성 결과는 로컬 스토리지, 원격 서버, 파일 등에 저장하지 않는다.

### 추첨 트리거

- 버튼 추첨은 모든 플랫폼에서 지원한다.
- 흔들기 추첨은 네이티브 환경에서 지원한다.
- 웹 환경에서는 흔들기 센서 의존 기능을 실행하지 않는다.
- 추첨 중에는 버튼 또는 흔들기 입력이 추가 추첨을 시작하지 않아야 한다.
- 연속 흔들기 입력으로 인한 중복 실행을 막기 위해 쿨다운을 적용한다.

### 애니메이션

- 핵심 비주얼은 "로또볼 추첨기" 콘셉트다.
- 추첨 시작 후 볼이 섞이는 연출을 보여준다.
- 번호 볼은 6개가 순차적으로 공개된다.
- 추첨 완료 상태에서는 모든 번호가 안정적으로 고정되어 보여야 한다.
- 애니메이션이 완료되기 전 화면 전환이나 재추첨이 발생해도 앱이 오류 상태에 빠지지 않아야 한다.

### 피드백

- 네이티브 환경에서는 번호 공개 또는 완료 시 햅틱 피드백을 제공한다.
- 햅틱이 지원되지 않거나 실패하더라도 추첨 플로우는 계속 진행된다.
- 사운드는 v1 범위에서 제외한다.

### 문구

- 기능 카드 제목: "로또 번호 추첨"
- 기능 카드 설명 예시: "흔들거나 버튼을 눌러 행운의 번호 6개를 뽑아보세요."
- 화면 제목 예시: "로또 번호 추첨"
- 기본 버튼 문구: "번호 뽑기"
- 추첨 중 버튼 문구 예시: "추첨 중..."
- 결과 후 버튼 문구 예시: "다시 뽑기"
- 안내 문구: "무작위 추천이며 당첨을 보장하지 않아요."

## 4. 화면 및 상태

### `/` 홈 기능 허브

- 기존 기능 허브의 카드 목록에 `/lotto`로 이동하는 새 카드가 추가된다.
- 기존 `/food` 카드의 동작은 변경하지 않는다.
- 카드 목록은 향후 기능 확장이 쉬운 배열 기반 구조를 유지한다.

### `/lotto` 로또 추첨 화면

- Stack 헤더를 사용한다.
- 초기, 추첨 중, 결과 표시 상태를 가진다.
- 주요 UI 구성은 다음을 포함한다.
  - 로또볼 추첨기 비주얼 영역
  - 추첨 시작/재추첨 버튼
  - 결과 번호 표시 영역
  - 당첨 보장 아님 안내 문구
  - 네이티브 환경의 흔들기 안내 문구

### 상태 정의

- `idle`: 아직 결과가 없고 추첨을 시작할 수 있는 상태
- `drawing`: 추첨 연출이 진행 중이며 추가 입력이 막힌 상태
- `result`: 6개 번호가 공개되어 다시 추첨할 수 있는 상태

## 5. 성공 기준

- 사용자는 `/`에서 "로또 번호 추첨" 카드를 통해 `/lotto`로 이동할 수 있다.
- 사용자는 버튼으로 추첨을 시작할 수 있다.
- 네이티브 환경에서는 흔들기로도 추첨을 시작할 수 있다.
- 추첨 중에는 중복 실행이 방지된다.
- 결과는 1부터 45까지의 중복 없는 숫자 6개다.
- 결과 번호는 추첨된 순서 그대로 표시된다.
- 결과에는 "무작위 추천이며 당첨을 보장하지 않는다"는 안내가 포함된다.
- v1에는 생성 기록, 즐겨찾기, 최신 당첨번호 조회, 당첨 비교 기능이 없다.

## 6. 테스트 계획

### 번호 생성 테스트

- 6개의 숫자가 생성되는지 확인한다.
- 모든 숫자가 1 이상 45 이하인지 확인한다.
- 한 결과 안에 중복 숫자가 없는지 확인한다.
- 반복 생성해도 범위와 중복 없음 조건이 유지되는지 확인한다.
- 최종 결과가 오름차순으로 강제 정렬되지 않고 추첨순을 유지하는지 확인한다.

### UI 플로우 테스트

- 초기 상태에서 추첨 버튼이 표시되는지 확인한다.
- 버튼을 누르면 추첨 중 상태로 전환되는지 확인한다.
- 추첨 중에는 버튼이 비활성화되거나 중복 입력이 무시되는지 확인한다.
- 추첨 완료 후 6개 번호와 안내 문구가 표시되는지 확인한다.
- 다시 추첨하면 새 번호가 생성되고 이전 결과가 대체되는지 확인한다.

### 플랫폼 조건 테스트

- 웹 환경에서는 버튼 추첨이 동작하고 흔들기 센서 로직이 실행되지 않는지 확인한다.
- 네이티브 환경에서는 버튼 추첨과 흔들기 추첨이 모두 동작하는지 확인한다.
- 햅틱 호출 실패 또는 미지원 환경에서도 추첨이 완료되는지 확인한다.

### 접근성 및 문구 테스트

- 추첨 버튼의 라벨이 현재 상태를 이해할 수 있게 표시되는지 확인한다.
- 결과 번호를 화면 읽기 도구가 의미 있게 읽을 수 있는지 확인한다.
- 당첨 보장 아님 안내 문구가 결과와 함께 노출되는지 확인한다.

## 7. 구현 제약 및 의존성

- 외부 API와 네트워크 요청을 사용하지 않는다.
- 로컬 저장소를 사용하지 않는다.
- 번호 생성은 순수 함수로 분리해 테스트 가능하게 만든다.
- 흔들기 감지는 네이티브 환경에서만 활성화한다.
- 햅틱은 지원 환경에서만 보조 피드백으로 사용하며 핵심 플로우의 성공 조건으로 삼지 않는다.
- 기존 `/food` 기능과 공통 앱 레이아웃 동작을 깨뜨리지 않는다.

## 8. 향후 확장 후보

- 마음에 드는 조합 즐겨찾기
- 여러 게임 동시 생성
- 최신 당첨번호 조회
- 생성 번호와 당첨번호 비교
- 공유하기
- 추첨 연출 테마 선택
