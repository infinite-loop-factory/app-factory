# 🍷 한 잔의 기록 — Product Requirements Document

> 내가 마신 술, 간 장소, 발견한 페어링을 모두 기록하는 나만의 음주 아카이브

---

## 1. 개요

### 1.1 프로덕트 비전

일상에서 마시는 모든 주류(위스키, 와인, 맥주, 사케, 칵테일 등)를 한 곳에서 기록하고, 장소와 페어링까지 연결해 나만의 음주 아카이브를 만드는 모바일 앱.

### 1.2 목표

- 주류 종류에 관계없이 통합된 하나의 흐름으로 테이스팅 기록
- 마신 장소, 방문한 바/증류소/와이너리를 지도 위에 시각화
- 음식 페어링 경험을 축적하고 다시 꺼내볼 수 있는 구조
- Expo SDK 54의 네이티브 기능을 적극 활용한 모바일 최적화 경험

### 1.3 타겟 사용자

- 주류를 즐기며 자신의 경험을 체계적으로 기록하고 싶은 사람
- 바, 레스토랑, 증류소 등 장소 탐방을 좋아하는 사람
- 음식과 술의 페어링에 관심 있는 사람

### 1.4 기술 환경

- Expo SDK 54 (React Native 0.81, React 19.1)
- New Architecture 기반
- iOS / Android 지원

### 1.5 데이터 아키텍처 — 풀 로컬

- 별도의 클라우드 백엔드(Supabase, Firebase 등) 없이 **expo-sqlite 기반 풀 로컬** 구조로 운영
- 모든 데이터(테이스팅 노트, 장소, 페어링, 뱃지, 설정)는 디바이스 내 SQLite DB에 저장
- 사진 파일은 expo-file-system을 통해 디바이스 로컬 디렉토리에 저장
- 간단한 설정값/UI 상태는 expo-sqlite의 localStorage API 활용
- 알림은 모두 expo-notifications 로컬 알림으로 처리 (서버 푸시 없음)
- 오프라인 환경에서도 모든 기능이 정상 동작
- 멀티 디바이스 동기화 및 클라우드 백업은 v1 범위에 포함하지 않음 (향후 필요 시 별도 설계)

## 2. 정보 구조 (Information Architecture)

```
App
├── 홈 (테이스팅 피드)
│   ├── 최근 기록 타임라인
│   ├── 빠른 기록 버튼 (FAB)
│   └── 검색 / 필터
├── 지도
│   ├── 내 기록 핀 (방문한 장소)
│   ├── 위시리스트 핀 (가고 싶은 장소)
│   ├── 카테고리 필터 (바 / 증류소 / 와이너리 / 브루어리 / 레스토랑)
│   └── 장소 상세 → 해당 장소의 기록 목록
├── 페어링
│   ├── 페어링 기록 목록
│   ├── 페어링 등록
│   └── 주류별 / 음식별 필터
├── 마이
│   ├── 통계 대시보드
│   ├── 컬렉션 / 뱃지
│   ├── 데이터 내보내기
│   └── 설정 (알림, 테마 등)
└── 기록 작성 (모달/풀스크린)
    ├── 사진 촬영 / 선택
    ├── 기본 정보 입력
    ├── 태그 입력
    ├── 위치 자동 태깅 + 수동 수정
    └── 점수 / 메모
```

## 3. 핵심 기능 상세

### 3.1 테이스팅 노트 (메인 기능)

모든 데이터의 시작점. 한 잔을 마실 때마다 하나의 기록을 남긴다.

**기록 항목:**

| 항목 | 필수 여부 | 설명 |
|------|----------|------|
| 사진 | 선택 | 라벨, 잔, 분위기 등 자유롭게 (복수 가능) |
| 주류 카테고리 | 필수 | 위스키 / 와인 / 맥주 / 사케 / 칵테일 / 기타 |
| 이름 | 필수 | 주류 이름 (자유 텍스트) |
| 점수 | 선택 | 5점 만점 (0.5 단위) |
| 태그 | 선택 | 자유 태그 (예: "셰리캐스크", "IPA", "내추럴와인", "드라이") |
| 메모 | 선택 | 자유 텍스트 테이스팅 노트 |
| 장소 | 자동 | 현위치 자동 태깅 + 수동 수정 가능 |
| 가격 | 선택 | 금액 + 단위 (잔/병) |
| 날짜 | 자동 | 작성 시점 자동 기록, 수동 수정 가능 |

**사용 흐름:**
1. FAB 버튼 탭 → 기록 작성 화면 진입
2. 카메라로 라벨 촬영 또는 갤러리에서 선택
3. 카테고리 선택 → 이름 입력
4. 위치 자동 감지 (변경 가능)
5. 점수, 태그, 메모 등 선택 항목 입력
6. 저장 → 햅틱 피드백과 함께 완료

**목록/검색:**
- 기본 정렬: 최신순 타임라인
- 필터: 카테고리, 점수 범위, 태그, 기간
- 검색: 이름, 태그, 메모 내 텍스트 검색

---

### 3.2 지도

내 음주 기록과 장소를 지도 위에 시각화한다. "바 헌터"와 "증류소 순례" 기능을 하나의 지도 탭에 통합.

**지도 위 핀 종류:**

| 핀 타입 | 아이콘/색상 구분 | 설명 |
|---------|----------------|------|
| 방문 완료 | 채워진 핀 | 테이스팅 기록이 있는 장소 |
| 위시리스트 | 빈 핀 | 가보고 싶은 장소 (기록 없음) |

**장소 카테고리:**
- 바 / 펍
- 증류소
- 와이너리
- 브루어리
- 레스토랑
- 기타

**주요 인터랙션:**
- 핀 탭 → 장소 요약 바텀시트 (장소명, 방문 횟수, 최근 기록 미리보기)
- 바텀시트 탭 → 장소 상세 (해당 장소에서의 모든 기록 목록)
- 카테고리 필터 토글 (지도 상단)
- 도시/리전별 방문 통계
- 위시리스트 장소 추가 (지도에서 롱프레스 또는 검색 후 추가)

**지오펜싱 알림:**
- 위시리스트에 등록한 장소 반경 접근 시 푸시 알림
- "근처에 가보고 싶었던 [장소명]이 있어요!"
- 알림 탭 시 해당 장소 상세로 이동

---

### 3.3 페어링

"이 술 + 이 음식 = 어땠다"를 기록하는 공간.

**페어링 기록 항목:**

| 항목 | 필수 여부 | 설명 |
|------|----------|------|
| 주류 | 필수 | 기존 테이스팅 노트에서 선택 또는 새로 입력 |
| 음식 | 필수 | 자유 텍스트 (예: "블루치즈", "스시 오마카세") |
| 사진 | 선택 | 페어링 사진 |
| 궁합 점수 | 필수 | 👎 별로 / 😐 보통 / 👍 좋음 / 🤩 최고 |
| 메모 | 선택 | 어떤 점이 좋았는지, 왜 잘 맞았는지 |
| 장소 | 자동 | 연결된 테이스팅 노트의 장소 또는 별도 입력 |

**목록/탐색:**
- 카드형 목록 (주류 사진 + 음식명 + 궁합 점수)
- 필터: 주류 카테고리별, 궁합 점수별
- 특정 주류의 페어링 모아보기
- 특정 음식의 페어링 모아보기 (역방향 탐색)

---

### 3.4 마이

개인 통계와 설정을 관리하는 공간.

**통계 대시보드:**
- 총 기록 수 / 이번 달 기록 수
- 카테고리별 비율 (도넛 차트)
- 평균 점수 추이 (월별 라인 차트)
- 방문 장소 수 / 도시 수
- 가장 많이 사용한 태그 TOP 5
- 페어링 기록 수 / 최고 궁합 조합

**컬렉션 & 뱃지 (게이미피케이션):**
- 카테고리별 기록 수 마일스톤 (예: 위스키 10종, 50종, 100종)
- 장소 관련 (예: 3개 도시 방문, 증류소 첫 방문)
- 페어링 관련 (예: 페어링 10개 기록)
- 뱃지 획득 시 햅틱 피드백 + 축하 모달

**데이터 관리:**
- JSON 내보내기 (전체 기록 → expo-sharing으로 공유)
- 사진 포함 로컬 백업
- 데이터 초기화

**설정:**
- 알림 on/off 및 시간 설정
- 지오펜싱 반경 설정 (기본 500m)
- 테마 (라이트/다크)
- 기본 통화 설정

## 4. 알림 설계

> 모든 알림은 expo-notifications의 **로컬 알림**으로 처리한다. 서버 기반 원격 푸시는 사용하지 않는다.

| 알림 종류 | 트리거 | 내용 |
|----------|--------|------|
| 기록 리마인더 | 매주 금요일 저녁 (설정 가능) | "이번 주에 마신 술, 기록해두세요 🍷" |
| 지오펜싱 | 위시리스트 장소 반경 진입 | "근처에 [장소명]이 있어요!" |
| 뱃지 획득 | 마일스톤 달성 시 | "위스키 10종 달성! 🏅" |
| 주간 리포트 | 매주 일요일 아침 | "이번 주 N잔 기록, 평균 점수 X.X점" |

## 5. 데이터 구조 (개념 수준)

### 5.1 저장소 전략

| 저장 대상 | 저장소 | 패키지 |
|----------|--------|--------|
| 테이스팅 노트, 장소, 페어링, 뱃지 | SQLite DB (`sip-note.db`) | expo-sqlite |
| 사진 파일 | 앱 Documents 디렉토리 | expo-file-system |
| 앱 설정, 최근 검색어, UI 상태 | localStorage (SQLite 기반) | expo-sqlite/localStorage |
| JSON 내보내기 파일 | 앱 Documents 또는 공유 디렉토리 | expo-file-system + expo-sharing |

- 사진은 DB에 파일 경로(URI)만 저장하고, 실제 파일은 로컬 디렉토리에 별도 관리
- 내보내기 시 JSON + 사진을 묶어 공유 시트로 전달

### 5.2 핵심 엔티티

**TastingNote (테이스팅 노트)**
- id, category, name, score, tags[], memo, price, priceUnit, date
- photos[] (로컬 파일 경로)
- placeId (장소 참조)

**Place (장소)**
- id, name, category, address, latitude, longitude
- isWishlist (위시리스트 여부)
- visitCount

**Pairing (페어링)**
- id, tastingNoteId (또는 별도 주류 정보), foodName
- matchScore, memo
- photo (로컬 파일 경로)
- placeId

**Badge (뱃지)**
- id, type, title, description, condition
- unlockedAt

### 5.3 관계

```
TastingNote ──N:1──▶ Place
Pairing ──N:1──▶ TastingNote (선택적)
Pairing ──N:1──▶ Place (선택적)
Badge ── 조건 기반 자동 해제
```

## 8. 단계별 구현 로드맵

### Phase 1 — 기록의 시작
> SQLite DB 초기화 + 테이스팅 노트 CRUD + 카메라 + 사진 로컬 저장

핵심 루프를 먼저 완성한다. DB 스키마 생성, 사진 찍고, 정보 입력하고, 저장하고, 목록에서 보는 것.

### Phase 2 — 장소와 연결
> 위치 자동 태깅 + 지도 탭 + 장소 관리

기록에 위치가 붙으면서 데이터가 공간적 맥락을 갖게 된다.

### Phase 3 — 경험 확장
> 페어링 기록 + 위시리스트 + 로컬 데이터 내보내기 (JSON + 사진)

기록의 범위를 음식 페어링까지 넓히고, 가고 싶은 장소를 관리한다. expo-file-system + expo-sharing으로 데이터 백업/공유.

### Phase 4 — 몰입 장치
> 지오펜싱 알림 + 리마인더 + 뱃지 시스템 + 통계 대시보드

백그라운드 기능과 게이미피케이션으로 지속 사용 동기를 만든다.
